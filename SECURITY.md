# 보안 가이드

## 구현된 보안 기능

### 1. 비밀번호 보안
- **Web Crypto API 사용**: PBKDF2 + SHA-256을 사용한 강력한 비밀번호 해싱
- **높은 반복 횟수**: 100,000회 반복으로 브루트포스 공격 방지
- **하위 호환성**: 기존 평문 비밀번호 자동 마이그레이션

### 2. 데이터 암호화
- **AES-GCM 암호화**: 민감한 사용자 데이터를 localStorage에 암호화하여 저장
- **세션 기반 키**: 브라우저 세션 기반 암호화 키 생성
- **자동 암호화/복호화**: 저장 및 로드 시 자동으로 암호화/복호화 수행

### 3. 입력값 검증
- **XSS 방지**: HTML 태그, JavaScript 코드 제거
- **입력값 정제**: 모든 사용자 입력값 검증 및 정제
- **형식 검증**: 이메일, 비밀번호, 사용자 이름 등 형식 검증

### 4. HTTPS 지원
- **자동 감지**: HTTPS 사용 여부 자동 감지
- **보안 경고**: HTTP 사용 시 경고 메시지 표시
- **개발 환경 예외**: localhost는 예외 처리

## 프로덕션 배포 시 주의사항

### 1. HTTPS 필수
프로덕션 환경에서는 반드시 HTTPS를 사용해야 합니다.

**배포 옵션:**
- **Vercel/Netlify**: 자동 HTTPS 제공
- **자체 서버**: Let's Encrypt로 무료 SSL 인증서 발급
- **Nginx/Apache**: 역방향 프록시 설정

### 2. 서버 사이드 보안 (권장)
현재는 클라이언트 사이드 보안만 구현되어 있습니다. 프로덕션 환경에서는 다음을 권장합니다:

- **서버 사이드 인증**: JWT 토큰 기반 인증
- **서버 사이드 비밀번호 해싱**: bcrypt 또는 argon2 사용
- **API 엔드포인트 보호**: CORS, Rate Limiting 설정
- **데이터베이스 암호화**: 민감 데이터 암호화 저장

### 3. 환경 변수 설정
민감한 설정값은 환경 변수로 관리하세요:

```env
VITE_API_URL=https://api.example.com
VITE_ENCRYPTION_KEY=your-secret-key
```

### 4. Content Security Policy (CSP)
`index.html`에 CSP 헤더를 추가하세요:

```html
<meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self' 'unsafe-inline';">
```

### 5. 정기적인 보안 업데이트
- 의존성 패키지 정기 업데이트
- 보안 취약점 스캔
- 로그 모니터링
